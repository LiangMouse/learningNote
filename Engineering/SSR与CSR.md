# CSR与SSR

## 概念
CSR(客户端渲染)与SSR(服务端渲染)是用户使用浏览器获取资源的两种方式, 客户端渲染也就是用户直接拿到残缺的HTML同时下载JS脚本动态下载DOM并渲染，数据由进一步的网络请求异步加载。而相对的，服务端渲染就是在服务端直接返回给客户端完整的HTML进行渲染

## 常见问题

大致从两者区别，合适场景，
### 两者的区别

CSR（Client-Side Rendering）：

- 浏览器下载 空 HTML，由 JavaScript 动态生成 DOM 并渲染。

- 数据通过 API 异步获取（如 fetch、axios）。

- 典型框架：React、Vue（默认模式）。

- 优点：交互性强，适合 SPA（单页应用），前后端分离开发方便。

- 缺点：首屏加载慢（需等 JS 执行完），SEO 不友好（爬虫难以抓取动态内容）46。

SSR（Server-Side Rendering）：

- 服务器生成 完整 HTML 并返回给浏览器，浏览器直接渲染。
- 数据在服务器端获取并注入 HTML。
- 典型框架：Next.js（React）、Nuxt.js（Vue）。
- 优点：首屏加载快，SEO 友好，适合内容型网站（如博客、电商）。
- 缺点：服务器压力大，交互性稍弱（需结合客户端 水合 hydration）

### 选择场景

由于SSR的首屏极致性能优化+SEO友好的特点，使得SSR特别适合做营销推广页。
而CSR由于现在也有很多性能优化解决方案+热更新开发的卓越体验，开发便利使得具体应用页使用CSR
