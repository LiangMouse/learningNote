# HTTP版本迭代升级

HTTP1.1修复了0.9和1.0的一些问题，作为第一个HTTP的标准版本在1997年问世，在之后2.0和3.0版本又做了一些迭代升级，之后让我们详细看看他们的差异

## HTTP1.1

优势
1. HTTP1.1默认支持长连接，TCP连接在全部请求结束后才会中断
2. 1.1通过管道进行多请求的并行传输

1. 管道接收响应顺序按照请求发出顺序，会造成乱序

HTTP 1.1 和 HTTP 2.0 之间有以下区别：

1. **多路复用**：HTTP 2.0 支持多路复用，即在一个 TCP 连接上可以同时传输多个请求/响应。`1.1`的TCP连接只能串行进行数据的传输以及发送和处理请求，倘若一个请求返回慢，就会造成类似**堵车**的情况。HTTP2新特性多路复用避免了 HTTP 1.1 的**队头阻塞**问题(前边的请求处理慢，后边的都会堵塞)，并且这个并行请求是可以**设置优先级**来控制响应顺序的，从而进一步提高了性能。

2. **二进制分帧**：HTTP 1.1 采用的是文本协议，而 HTTP 2.0 则采用了二进制分帧，将请求和响应数据分割成二进制的数据帧，在传输时更加高效。

3. **首部压缩**：HTTP 2.0 引入了新的首部压缩算法 `HPACK`，可以将 HTTP 首部信息进行压缩，减少数据传输量，提高性能。

4. **服务器主动推送**：HTTP 2.0 支持服务器主动推送，即在客户端发送请求之前，服务器可以将一些预测到客户端需要的资源主动推送给客户端，减少请求延迟，提高页面加载速度。

总之，HTTP 2.0 比 HTTP 1.1 在传输效率和性能方面有了很大提升，并在一些方面引入了新的特性，为 Web 应用的发展提供了更好的支持。
