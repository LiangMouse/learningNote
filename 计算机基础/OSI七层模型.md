# OSI 七层模型

| TCP/IP 层级   | OSI 参考模型层级 | 协议及技术                   |
| ----------- | ---------- | ----------------------- |
| 应用层         | 应用层        | HTTP, FTP, SNMP, DNS    |
|             | 表示层        | JPEG, ASCII, DECOIC, 加密 |
|             | 会话层        | SSL, TLS                |
| 传输层         | 传输层        | TCP, UDP                |
| 网络层         | 网络层        | IP, ICMP, ARP, RIP 等    |
| 网络接口层/数据链路层 | 数据链路层      | FDDI, Ethernet, PPP     |
|             | 物理层        | IEEE 802.1              |

---

- 物理层

  物理层直接与物理介质打交道，最重要的是信道。有限信道(电缆、光纤)，无线信道(无线电波)，获取传送信号转换成二进制**比特流**

- 数据链路层

  比特流组装，网卡对比特流进行分组成**数据帧**并拼装上具有唯一标识的 **MACId**.拼装后知道给谁发，然后进行广播，给目标对象发送。

- 网络层

  在这一层定义**IP**，MAC 并非唯一，使用 IP 进行唯一的**寻址**，使用路由器端对端的方式进行寻找

- 传输层
  定义端口号(比如:8080,:443 这些)，控流以及校验以及**TCP**和**UDP**协议，这一层又叫数据段，在 OSI 中起承上启下的作用

  简单说，TCP 面向连接并且可靠，会进行三次握手四次挥手，但会降低速度。UDP 没有三次握手四次挥手，不稳定，但速度快，实时性好，适用于直播游戏

- 会话层

  发送方和接收方进行通信时创建、维持、之后断开连接的地方，有检查点的机制维持可靠会话。这一层被叫报文

- 表示层

  对图片，音乐，ASCII 等内容进行编码解码成二进制，"翻译官"，也属于报文内容

- 应用层

  开发中使用最多的一层，AJAX 调接口发送 http 请求，域名系统 DNS，webSocket 长连接，SSH 协议，这一层也属于报文
