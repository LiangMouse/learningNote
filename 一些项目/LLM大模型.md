## 技术栈
**NextJs**
njs的路由选择是在app文件夹下,新建一个文件夹a，a下有一个命名为page.jsx的文件,在这个nextjs的应用下输入/a，则会渲染./a/page.jsx文件
1. **文件路由系统**：`app` 目录中的文件和文件夹直接映射到应用的路由。每个子目录可以代表一个路由，文件名通常与路由路径相关联。
2. **布局和嵌套布局**：你可以在 `app` 目录中定义全局布局和嵌套布局，允许不同的页面共享布局组件。
3. **服务器组件**：Next.js 13 引入了 React 服务器组件的支持，`app` 目录鼓励使用服务器组件来优化性能。
4. **简化结构**：通过使用 `app` 目录，Next.js 提供了一种更简洁和模块化的方式来组织应用的结构，而不需要额外的配置。
`layout.tsx` 是当前目录及其子目录页面的**父级容器**，它会包裹所有子页面的内容。
