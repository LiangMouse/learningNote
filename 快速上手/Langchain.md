# Langchain 快速上手

WIP

开源的AI 流程编排库，支持且仅支持Python和JavaScript的SDK使用，支持绝大部分LLM的通用接口，因此就能有个集成环境与项目业务数据结合
## 组成
Langchain通过抽象简化了LLM应用程序编程，抽象也就是封装，我们可以把WorkFlow的不同节点在Langchain上的组件进行抽离使用，但不需要知道其实际实现，其中组件的prompt,LLM选型都是可以自定义的。

- 当WorkFlow需要访问内部信息，如文档时，Langchain称其为Index索引。提供不同的Loader以支持解析的实现，本质是文件存储服务中导入数据源，进行向量嵌入把数据表示为一系列向量点。并提供文本分割器把文本 `text spliting` 切成保持语义化的最小单元—— `Chunk`。已根据方法选型进行向量化。
- memory 本质上，大模型不具备保留上次对话信息的能力，Langchain通过在应用中添加内存解决这个问题，内存长度可以自定义如整个会话或会话摘要
- agents 输入包括可使用的工具列表，用户prompt或查询输入
## 使用场景

- 众所周知，不同大模型的专长是不一样的，例如有些支持超长上下文，有些图像等多模态支持给力，有些则是很实在的便宜。在一个应用中不同场景使用不同的大模型API就是Langchain适合的场景之一

## 举例

### chatbot

- 特定上下文。LangChain可为聊天机器人的特定用途提供适当的上下文
- 文本预处理能力。对PDF，成绩单，图表等通过LLM等能力转成他所需要的信息
- 知识库。使用特定温度或专业知识库，LLM从存储中检索相关信息帮助输出信息
