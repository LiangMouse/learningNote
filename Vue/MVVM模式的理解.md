# MVVM 模式的理解

## MVVM 模型的概念

![MVVM结构](/MVVM模式的理解01.png)
MVVM（Model-View-ViewModel）是一种架构模式，用于分离应用程序的 UI 界面和业务逻辑。它通过数据绑定（data binding）和响应式机制来自动更新界面，从而简化了前端开发流程。

Model（模型）：用于存储应用的数据和业务逻辑，通常包括从服务器获取的数据或处理后的业务数据。它与视图无直接关联。
View（视图）：用户界面，直接展示给用户的内容，通常是 HTML/CSS 等。
ViewModel（视图模型）：连接 View 和 Model 的桥梁，负责处理数据和业务逻辑的转换，提供给视图更新展示的数据，并通过数据绑定将视图的用户交互同步到模型。

## MVVM 的核心机制 - 数据绑定

MVVM 的一个核心特点是数据绑定。通过双向数据绑定，视图会自动根据 Model 的数据变化更新，而不需要手动操作 DOM。同时，View 的更新也会同步回 Model，这样实现了视图和数据的双向同步。
