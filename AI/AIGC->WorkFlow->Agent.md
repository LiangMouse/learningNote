# Aigc->WorkFlow->Agent

## LLm 基础

- LLM 对个人等场景下的专有隐私信息了解有限
- LLM 是被动的，必须被动接受信息输入才能做响应，无法主动给自己下任务

## WorkFlow

在如上的场景基础下，如果提前告知 LLM“当被问到个人规划问题前，请先检索日历应用信息” 她就能够执行比如“请问我下一次约会的时间是什么时候”这种问题，但是倘若场景再次更迭，就得编排更多岔路的 WorkFlow，比如明天天气要去搜索天气软件信息等。

WorkFlow 遵循预定义的路径，该路径可能涉及到从外部工具进行检索

WorkFlow 只能遵循开发者给定的工作流程

从本质上讲，RAG（检索增强生成）也只是在最终生成前的一个 workflow 的 step 编排

类比我们做一件事情，比如睡醒到出门，workflow 就是在起点终点之间编排路径

这里可以在https://cloud.dify.ai/app/1ee558ae-33ac-4058-96c3-d4c1d55403b3/workflow dify 这个平台上搭建工作流试用下~还是很直观的，目前业内也有用 dify 等搭建 AI 可视化低代码平台的尝试

## Agent

Agent 的本质就是把 workflow 的制定者给到 LLM 自己。从定义上，AI 智能体是自主感知环境、做出决策并执行行动以实现特定目标的智能实体，核心组成部分是：规划（Planning）、记忆（Memory）、工具使用（Tool Use）。

因此 Agent 的最常见配置就是 ReAct => Reasoning + Act （推理实现路径以及实现）

具备观察中间态输出是否符合预期并判断是否需要重新迭代的能力
