# 箭头函数和普通函数的区别

箭头函数的设计目的是简化函数声明，并且继承了外部作用域的 this 和 arguments。这是他的根本不同
对于箭头函数个体考量，来具体说明两者的不同

1. 在语法上，普通函数使用`function`关键字，箭头函数使用`=>`关键字声明
2. 箭头函数没有自己的`this`,他会捕获自己在定义时所处的外部执行环境的`this`,所以箭头函数中`this`指向从被定义就已经确认，之后不会改变(因此 call/apply/bind 方法不会改变其 this 指向)
3. 箭头函数由于没有自己的`this`，这与构造函数的`new`
   过程冲突，因而**不能作为构造函数使用**
4. 箭头函数没有自己的`arguments`对象，箭头函数的`arguments`其实是外层函数执行环境的值
5. 箭头函数没有原型`prototype`。普通函数原型用于存放该函数作为构造函数时，所创建的实例对象共享的方法和属性。

```javascript
let sayHi = () => {
  console.log("Hello World !");
};
console.log(sayHi.prototype); // undefined
```
