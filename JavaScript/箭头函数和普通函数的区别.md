# 箭头函数和普通函数的区别

箭头函数的设计目的是简化函数声明，并且继承了外部作用域的 this 和 arguments。这是他的根本不同
对于箭头函数个体考量，来具体说明两者的不同

1. 在语法上，普通函数使用`function`关键字，箭头函数使用 `=>` 关键字声明
2. 箭头函数没有自己的`this`.它会像查找一个普通变量一样，沿着词法作用域链向上查找，直到找到一个包含 this 的上下文为止。而我们常说的`this`是函数在 Js 引擎编译后产生的上下文中的一部分，而箭头函数是缺失的。
3. 箭头函数不能充当构造函数。由于没有自己的`this`，这与构造函数的`new`
   过程冲突，因而**不能作为构造函数使用**
4. 箭头函数没有自己的`arguments`对象。箭头函数的`arguments`其实是外层函数执行环境的值
5. 箭头函数没有原型`prototype`。这也与他不能做构造函数是一致的
