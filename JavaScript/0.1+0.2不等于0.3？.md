# 0.1+0.2 不等于 0.3

打开 F12 控制台输入 0.1+0.2 回车会发现结果并非 0.3 而是`(0.1+0.2) => 0.30000000000000004`

为什么?

浮点数精度问题。`JavaScript`使用`IEEE754`来表示数字。例如 0.1 需要先用乘二取余法转换成二进制表示。但并不是所有十进制小数都能转换成“整”的二进制小数，此时他是一个无限循环小数`0.000110011001100110011...`由于计算机的存储位数有限，这些数字会被截断，导致计算机存储值不完全等于 0.1.

0.2 也是这样，此时两数相加，误差累计，导致结果并非`0.3`


## 如何解决？

- `Number.toFixed`方法可以截取给定位数的小数，保证精度
- 使用 `Math.round()`方法

  `console.log(Math.round((0.1 + 0.2) * 100) / 100); // 输出 0.3`
  进行四舍五入
