# null 和 undefined 的区别

### **`null` 和 `undefined` 的含义**

---

1. **`null`**

   - **含义**：表示一个**空值**，通常用于手动赋值，表示“没有值”或“值为空”。
   - **特点**：
     - 值是一个对象类型。
     - 常用来表示明确意图上的空值，例如某变量的值尚未被指定。
   - **示例**：
     ```javascript
     let obj = null; // 显示地表示对象没有值
     console.log(obj); // null
     ```

2. **`undefined`**

   - **含义**：表示一个**未定义的值**。
   - **特点**：
     - 是一个原始类型，表示变量已声明但尚未赋值。
     - JavaScript 中许多场景下会自动返回 `undefined`。
   - **常见场景**：
     - 声明未赋值的变量。
     - 对象中访问不存在的属性。
     - 函数未返回值。
   - **示例**：

     ```javascript
     let x; // 声明未赋值
     console.log(x); // undefined

     let obj = {};
     console.log(obj.name); // undefined
     ```

---

### **区别**

| **特性**     | **`null`**                         | **`undefined`**                                    |
| ------------ | ---------------------------------- | -------------------------------------------------- |
| **定义**     | 明确表示“没有值”                   | 表示“未定义”                                       |
| **类型**     | `object`（历史遗留问题）           | `undefined`                                        |
| **赋值方式** | 通常由开发者显式赋值               | 通常是变量未赋值、系统自动赋值                     |
| **用途**     | 用于手动清空对象或值               | 表示变量尚未初始化、值未知或不存在                 |
| **常见场景** | - 数据库字段为空<br>- 函数参数空值 | - 声明未赋值<br>- 函数无返回值<br>- 对象属性不存在 |

---

### **相等与全等运算的差异**

- **相等运算（`==`）**：`null` 和 `undefined` 会被认为相等。
  ```javascript
  console.log(null == undefined); // true
  ```
- **全等运算（`===`）**：`null` 和 `undefined` 类型不同，因此不相等。
  ```javascript
  console.log(null === undefined); // false
  ```

---

### **总结**

- 使用 `null` 表示某变量**主动清空或无值**。
- `undefined` 常用于**系统自动赋值**或表示未定义的状态。
