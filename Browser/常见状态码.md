# 常见 HTTP 状态码

## 概述

1xx: 协议处理的中间状态，还在**请求中**
2xx: 报文已收到并成功处理
3xx: 重定向，资源发生变动，需要重新发送请求
4xx: 客户端错误，请求报文有误，服务器无法处理
5xx: 服务器处理请求时发生错误

### 常见状态码

- 200 - 成功。
- 201 - 创建用户成功
- 204 - 无响应体的请求正确响应
- 206 - 部分资源已正确处理（常用于断点续传场景中）
- 301 - 永久重定向（配合 location，浏览器自动处理）。
- 302 - 临时重定向（配合 location，浏览器自动处理）。
- 304 - 协商缓存（重定向到本地资源）
- 400 - 客户端请求的语法错误
- 401 - 身份认证有问题
- 403 - 没权限。
- 404 - 资源未找到。
- 409 - 编辑冲突问题
- 500 - 服务端通用错误码。
- 502 - 上层网关报错
- 503 - 服务端超载或者维护状态
- 504 - 网关超时。

### 一些笑话

- 不要说「收到」，试着说 `200 OK`
- 不要说「不在」，试着说 `404 NOT FOUND`
- 不要说「拒绝」，试着说 `403 Forbidden`
- 不要说「不合适」，试着说 `505 error`
- 不要说「要被玩坏了」，试着说 `500 Internal Server Error`
- 我们不说「v 我 50」，但可以说 `402 Payment Required`
- 不要说「滚那边去」，试着说 `301 Permanent moved`
- 不要说「好舒服」，试着说 `100 Continue`
- 不要说「找他去」，试着说 `308 Permanent Redirect`
- 不要说「想喝茶」，试着说 `418 I'm a teapot`
- 不要说「我愿意」，试着说 `202 Accepted`
- 不要说「我们不合适」，试着说 `303 See Other`
- 不要复读，试着说 `304 Not Modified`
- 不要说「不可以」，试着说 `401 Unauthorized`
- 不要说「品味不够」，试着说 `101 Upgrade`
- 不要说「c 你吗这什么文件（`*.000`、`*.001` ......）啊怎么打不开，不想给就直说」，试着说 `206 Partial Content`
- 不要说「这不是你的错」，试着说 `502 Bad Gateway`
- 不要说「啊~好大...全都进来了」，试着说 `413 Request Entity Too Large`
