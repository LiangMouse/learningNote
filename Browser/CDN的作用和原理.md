# CDN 的作用和原理

`CDN`（Content Delivery Network，内容分发网络）是通过将内容分发到离用户更近的节点服务器来加快内容传输的网络架构。它能显著提升访问速度并降低延时，解决用户和源服务器距离过远的问题。

### 分发内容

CDN 支持静态和动态内容的分发。

- **静态内容**：长时间保持不变的资源（如图标、CSS、JS 等），可在 CDN 节点缓存，用户重复请求时直接从缓存返回，加快加载速度。
- **动态内容**：如用户交互生成的动态数据，通常难以缓存。部分 CDN 提供接口，可以部署代码，进行数据处理和返回

CDN 源服务器通过 `Cache-Control` 等 HTTP 头设置缓存策略，控制资源的保存和失效时间。

### 安全性

CDN 在用户和源服务器间构建了一道“防护墙”，提升源站的安全性。

- **DDoS 防护**：CDN 提供分布式架构，能将攻击流量分散至多个节点，有效减轻源站压力。
- **负载均衡与任播**：CDN 会监控节点流量负载，自动将流量分配至负载较轻的节点，同时通过任播（Anycast）实现最近节点响应，提升请求速度。
- **TLS 安全传输**：CDN 对数据传输使用 TLS 加密，防止数据被窃取或篡改。

### CDN 其他优势

- **文件压缩与优化**：CDN 支持对文件进行压缩（如 GZIP、Brotli），去除不必要的空格和注释，减少传输数据量。
- **减少网络带宽开销**：通过缓存热点内容和减少源站请求频次，CDN 有效降低了源站的带宽成本。
