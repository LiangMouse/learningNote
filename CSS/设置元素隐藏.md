# 设置元素隐藏

以下是四种常见的 CSS 元素隐藏方式及其对比：

1. ​​display: none​​
​​**原理​**​：
完全从文档流中移除元素，不占据任何空间，如同元素不存在。
​​**特点**​​：
隐藏后元素无法接收事件（如点击、键盘输入）。
子元素也会被递归隐藏。
会导致页面重排（Reflow），影响性能。
​​使用场景​​：
完全不需要元素参与布局时（如切换显示内容、表单验证错误提示）。

2. ​​visibility: hidden​​
​**​原理**​​：
元素不可见，但仍保留原有的空间和布局位置。
​​**特点​**​：
元素无法接收事件（与 display: none 类似）。
子元素默认继承隐藏，除非显式设置 visibility: visible。
不会触发布局变化（No Reflow）。
​​**使用场景​**​：
需要临时隐藏元素但保留其占位空间的情况（如加载动画前的占位符）。

3. ​​opacity: 0​​
​​**原理​**​：
将元素透明度设为 0，使其完全透明，但仍占据空间并保留所有事件响应。
​​**特点**​​：
元素可交互（如点击、悬停）。
支持过渡动画（如淡入淡出效果）。
可能触发 GPU 加速，性能较好。
​​**使用场景​**​：
需要动画过渡或保持交互性的隐藏（如模态框关闭时的渐隐效果）。

4. ​​位移隐藏（如 position: absolute + 超出视口）​​
​​**原理**​​：
通过绝对定位将元素移出可视区域（如 left: -9999px），使其肉眼不可见。
​​**特点**​​：
元素仍在文档流中（若未脱离定位），可能影响布局。
可触发事件（如点击、键盘输入）。
可能影响可访问性（屏幕阅读器仍可能读取内容）。
​​**使用场景**​​：
需要隐藏元素但仍需保留其功能（如隐藏但可操作的按钮，或 SEO 优化隐藏内容）。
