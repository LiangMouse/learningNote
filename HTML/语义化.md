# HTML 语义化

HTML 语义化是指使用具有明确含义的 HTML 标签来构建网页，使得网页结构清晰，内容与标签的语义相匹配。简单来说，就是用正确的标签做正确的事。

## 为什么要使用语义化标签？

在过去，我们常常使用`<div>`和`<span>`来布局整个页面，但这会导致页面结构混乱，难以维护。而语义化标签则带来了许多优势：

1.  **提升代码可读性和可维护性**：语义化的 HTML 结构清晰，即使没有 CSS 样式，也能很好地呈现内容结构，方便团队协作和后期维护。
2.  **利于 SEO（搜索引擎优化）**：搜索引擎的爬虫能更好地理解网页的结构和内容，从而给予更高的权重，提升网站排名。
3.  **提升用户体验**：例如`title`、`alt`等属性可以为视障用户提供额外信息，屏幕阅读器可以根据标签的语义来朗读内容。
4.  **更好的设备兼容性**：语义化标签有助于不同设备（如手机、平板、阅读器）以更合适的方式展示页面内容。

## 常用语义化标签解析

HTML5 引入了许多新的语义化标签，以下是一些常用的标签：

- `<header>`：定义文档或节的页眉，通常包含标题、logo、导航等。
- `<footer>`：定义文档或节的页脚，通常包含作者信息、版权信息、联系方式等。
- `<nav>`：定义导航链接的部分。
- `<main>`：定义文档的主要内容，一个文档中只应有一个`<main>`标签。
- `<article>`：定义独立的、完整的内容块，如一篇博客、一篇文章、一条评论等。
- `<section>`：定义文档中的一个章节或区域，通常会包含一个标题。
- `<aside>`：定义与主要内容相关但可以独立存在的内容，如侧边栏、广告、引述等。

## 代码示例对比

### 语义化前（滥用 `div`）

```html
<div id="header">
  <h1>网站标题</h1>
  <div id="nav">
    <ul>
      <li><a href="#">首页</a></li>
      <li><a href="#">关于我们</a></li>
    </ul>
  </div>
</div>

<div id="main">
  <div class="article">
    <h2>文章标题</h2>
    <p>文章内容...</p>
  </div>
</div>

<div id="footer">
  <p>版权所有 &copy; 2024</p>
</div>
```

### 语义化后

```html
<header>
  <h1>网站标题</h1>
  <nav>
    <ul>
      <li><a href="#">首页</a></li>
      <li><a href="#">关于我们</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>文章标题</h2>
    <p>文章内容...</p>
  </article>
</main>

<footer>
  <p>版权所有 &copy; 2024</p>
</footer>
```

通过对比可以明显看出，使用语义化标签的 HTML 结构更加清晰、直观。
