# Git pull 和 fetch 的区别

`git pull`等价于`git fetch`+`git merge`

---

git fetch: 将远程仓库的最新更改下载到本地，但不会自动合并到当前分支

更新的远程追踪分支(本地对远程分支的引用,只读分支 Origin/master)，本地分支不会产生影响，适用于**查看远程分支变化后再决定**是否合并。

查看远程更新内容但不希望立即合并(手动检查和合并)

---

git pull: 自动执行 git fetch 和 git merge,将远程分支的更新拉取合并到当前分支

如果有冲突，必须解决冲突后才能完成合并。适用于快速获取并同步代码的场景
